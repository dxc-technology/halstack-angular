This component has no props. It has children and it follows the design guidelines available .

Everything in this component will be displayed as a header, at the top of the screen. If you want to show a , as a shortcut, you can directly use it without DxcApplicationLayoutHeader. This is optional and if it is not specified, the DxcHeader will be shown by default.

Everything in this component will be displayed as content inside a sidenav at the left. This is optional and if not specified, the sidenav will not be shown.

Name Default Description mode: 'overlay' | 'push' 'overlay' Default action over the content of the page, overlay the content or push to the right. In lower resolutions the mode will always be overlay. displayArrow: boolean true If false, the arrow button is hidden. In lower resolutions the arrow will be always displayed. padding: string | object Size of the padding to be applied to the custom area ('xxsmall' | 'xsmall' | 'small' | 'medium' | 'large' | 'xlarge' | 'xxlarge'). You can pass an object with 'top', 'bottom', 'left' and 'right' properties in order to specify different padding sizes.

This component has to be a child of DxcApplicationLayoutMain. Everything in this component will be displayed as a footer, at the bottom of the screen. If you want to show a , as a shortcut, you can directly use it without DxcApplicationLayoutFooter. This is optional and if it is not specified, the DxcFooter will be shown by default.

Everything in this component will be displayed as content inside the main container at the center.
